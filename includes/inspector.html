Inspector

The inspector allows you to examine the elements of a native app in the same style as you would with i.e. Firebug or the Chrome inspector.

To access the inspector, set a breakpoint in the test code and execute it in Debug mode.

When the application has stopped there, open a browser and got to http://localhost:5555/inspector/ ( change the port to match the one you started your server on )
You will see the app and display its object tree.

Mouse over the element you're interested in to find its attributes and be able to create locators.

If you hit the ESC. button, an xpath helper will appear, giving you access to real time validation of your xpath locators.

Working with localized apps

The inspector also allows you to inspect the internals of an app, and access its content files. Instead of using the content directly, you can use the key the content was generated from.The tests using the keys will be more robust as content change will not impact them.

When an element's key is found by ios-driver, the inspector will turn blue instead of the normal yellow, and some extra information will be displayed on the right hand side. The 'key' attribute can by used to create localization ready locators.
