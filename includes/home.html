ios-driver

Test any IOS native, hybrid, or mobile web application using Selenium / WebDriver.

About

Automate any IOS native, hybrid, or mobile web application using the Selenium / WebDriver API. ios-driver is fully compatible with the Selenium / Webdriver API. IOS automation is therefore as easy as automation for a browser.
ios-driver fully integrates with Selenium Grid so you can reuse your existing web automation infrastructure including your helper and utility classes (i.e. data creation, page objects etc.). .

Features

  • Implements the JSON wire protocol
  • Runs on emulators and devices
  • Can be run as a regular node in a Selenium Grid
  • Can be used in an existing Selenium Grid to manage device / simulator farm
  • First class support for localized native apps (same test works for all the localized versions )
  • No need to change the source
  • Off the shelf support for ios devices
  • No need to jailbreak the device
  • No additional apps needed on the device

Real Device vs Simulator

Using Simulator is the default way to use ios-driver.It offers more flexibility, and it is faster that tests on real device.However, if your specific needs require tests on real device, ios-driver also supports devices without having to make any modifications!

Scaling

ios-driver has been built with scaling in mind. GUI based tests are slow by nature slow and if run sequentially take too long to provide fast feedback. By following the Selenium / webdriver protocol, ios-driver can be used as a Selenium grid node, and the tests can be run in parallel on the same architecture as for the web.
